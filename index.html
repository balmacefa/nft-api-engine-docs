    Introduction ðŸ¤´âš”

    This is a fully automated solution ðŸ¤–ðŸ§ž that allows you to deploy an NFT with a single API call to the Polygon Matic blockchain ( this is a layer 2 Ethereum network). Other blockchains will be added in the future ( BSC, CELO, ONE).

    Key points ðŸ¥‡ðŸ”‘

    • Jobs are placed in a resilient Redis queue.

    • Deploy a smart contract for each collection

    • Uploads base64 data to IPFS (The InterPlanetary File System)

    • Create the NFT Json Metadata and include the IPFS results in it, as specified by your custom path. (Internally is done by lodash _.set)

    • Upload the compiled NFT Json Metadata to IPFS

    • Deploy the NFT to the blockchain

    • Use a webhook to notify you of the stages and status of the job.

    Why use a job queue ðŸ¤”â›‘

    Because of the nature of these networks and technology; There are multiple failure points related to network congestion, timeouts, incompletion, or erroneous results either on Blockchain or the IPFS. That is why we set up a resilience work queue, which attempts each step until the NFT is successfully deployed. To ensure that the NFT is deployed, jobs are retried 10 times until the error status is marked as unrecoverable.

    The Kittn API uses the following error codes:

    Error Code Meaning
    400 Bad Request -- Your request is invalid.
    401 Unauthorized -- Your API key is wrong.
    403 Forbidden -- The kitten requested is hidden for administrators only.
    404 Not Found -- The specified kitten could not be found.
    405 Method Not Allowed -- You tried to access a kitten with an invalid method.
    406 Not Acceptable -- You requested a format that isn't json.
    410 Gone -- The kitten requested has been removed from our servers.
    418 I'm a teapot.
    429 Too Many Requests -- You're requesting too many kittens! Slow down!
    500 Internal Server Error -- We had a problem with our server. Try again later.
    503 Service Unavailable -- We're temporarily offline for maintenance. Please try again later.
